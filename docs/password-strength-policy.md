# 회원가입 비밀번호 강도 정책 (4단계)

## 목적

- 회원가입 비밀번호를 `위험/보통/안전/매우 안전` 4단계로 판정한다.
- 프론트 UI 표시는 안내용이며, 최종 가입 허용/차단은 백엔드 판정 결과를 단일 기준으로 강제한다.
- 프론트와 백엔드는 **동일한 판정 기준**을 공유해야 하며, 기준 변경 시 동시에 반영한다.

## 기존 정책과 강도 판정의 관계

아래 기존 정책을 먼저 검증한다.

- 길이: 8자 이상 20자 이하
- 문자 조합: 영문/숫자/특수문자 중 2종 이상
- 금지 패턴: 공백, 3연속 반복, 아이디/이메일/전화번호 일부 포함, 4자 이상 연속 증가/감소

기존 정책 위반 시 즉시 가입 차단한다. (`AUTH4006`, `AUTH4007`, `AUTH4008`)
기존 정책 통과 후 강도 점수를 계산하고 4단계로 분류한다.

## 강도 점수 기준

### 길이 점수

- 10자 이상: +1
- 12자 이상: +2
- 16자 이상: +3

### 문자 조합 점수

- 1종: 0점 (길이와 무관하게 최종 `위험`)
- 2종: +1
- 3종: +2

### 최종 등급

- 0~1점: `위험` (가입 차단)
- 2~3점: `보통` (가입 허용)
- 4점: `안전` (가입 허용)
- 5점 이상: `매우 안전` (가입 허용)

## 응답 정책

### 기존 정책 위반

- 길이 위반: `AUTH4006`
- 조합 위반: `AUTH4007`
- 금지 패턴 위반: `AUTH4008`

### 강도 위험 등급

- 에러 코드: `AUTH4009`
- 메시지: `비밀번호 보안 강도가 위험 등급입니다.`

## 판정 예시

- **위험 (DANGEROUS)**
  - 예시: `Qa1x2w3e`
  - 산식: 길이(0점) + 조합(1점) = 총 1점
- **보통 (NORMAL)**
  - 예시: `Normal12ab`
  - 산식: 길이(1점) + 조합(1점) = 총 2점
- **안전 (SAFE)**
  - 예시: `C0mp!exAlpha9`
  - 산식: 길이(2점) + 조합(2점) = 총 4점
- **매우 안전 (VERY_SAFE)**
  - 예시: `T9!mQ2#vL7@rN5$wX`
  - 산식: 길이(3점) + 조합(2점) = 총 5점
